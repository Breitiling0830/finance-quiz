# 把刷題網站部署到 GitHub Pages 完整教學

完成後你的網址會是：`https://你的帳號.github.io/finance-quiz/`

預估時間：**5～10 分鐘**

---

## 你需要先準備好的檔案

我已經幫你放在「working folder」裡了，等下會用到：

1. `index.html` ← 主要的刷題頁面（已從 finance_quiz.html 改名）
2. `README.md` ← repo 介紹檔
3. `部署教學.md` ← 就是你正在看的這份

> 重點：檔案一定要叫 `index.html`，GitHub Pages 才會自動把它當成首頁。

---

## Step 1：建立新的 Repository（倉庫）

1. 登入 [https://github.com](https://github.com)
2. 點右上角的 **「+」** 按鈕 → 選 **「New repository」**
3. 填寫設定：
   - **Repository name**：`finance-quiz`（這就是網址裡的部分）
   - **Description**（選填）：`財管刷題網站`
   - **Public / Private**：選 **Public**（公開）
     > 重要：免費帳號的 GitHub Pages 只能用在 Public repo
   - **Add a README file**：⚠️ **不要勾**（我們自己要上傳一個）
   - 其他保持預設
4. 點下方綠色按鈕 **「Create repository」**

---

## Step 2：上傳檔案

剛建立完 repo 你會看到一個提示頁面。

1. 點選頁面中間的連結 **「uploading an existing file」**
   （或網址手動輸入：`https://github.com/你的帳號/finance-quiz/upload/main`）

2. 你會看到一個拖曳區域（"Drag files here..."）

3. **同時把這 3 個檔案拖進去**：
   - `index.html`
   - `README.md`
   - `部署教學.md`（選擇性，不放也沒關係）

4. 等上傳轉圈圈跑完（綠色勾勾出現）

5. 拉到頁面最下面的 **「Commit changes」** 區塊：
   - 第一個欄位輸入：`Initial commit`（或留空也行）
   - 下方選 **「Commit directly to the main branch」**
   - 按綠色按鈕 **「Commit changes」**

✅ 上傳完成！你應該會看到 repo 首頁列出 `index.html` 和 `README.md`。

---

## Step 3：開啟 GitHub Pages

1. 在 repo 頁面，點上方的 **「Settings」**（齒輪圖示）

2. 在左側選單找到並點 **「Pages」**

3. 設定 Source：
   - **Source** 下拉選 → **「Deploy from a branch」**
   - **Branch** 下拉選 → **「main」**
   - 旁邊那個資料夾下拉保持 **「/ (root)」**
   - 按 **「Save」**

4. 等 1～3 分鐘讓 GitHub 部署
   - 可以在同一頁面上方看到部署狀態
   - 等出現綠色框框寫：
     > ✅ Your site is live at `https://你的帳號.github.io/finance-quiz/`

5. 點那個網址就能打開你的刷題網站！🎉

## 之後想更新內容怎麼辦？

如果你之後改了題目或想換新的 HTML 檔：

1. 進入 repo 頁面
2. 點 `index.html` 檔案
3. 點右上角的鉛筆圖示 → 直接編輯
   或：點 **「Add file → Upload files」** 重新拖一個新的 `index.html` 上去蓋掉舊的
4. 拉到下方 **「Commit changes」**
5. 等 1～2 分鐘 GitHub Pages 會自動更新

## 常見問題

**Q1：我打開網址是 404 / 找不到頁面？**
- 確認檔名是 `index.html`（小寫，不是 Index.html 或 index.HTML）
- 確認 Pages 設定的 Branch 是 `main`，資料夾是 `/ (root)`
- 等 3～5 分鐘再重新整理（第一次部署比較慢）

**Q2：可以改網址嗎？**
- 可以！把 repo 重新命名（Settings → 最下面 Rename）即可
- 例如改成 `cpa-quiz`，網址就會變 `https://你的帳號.github.io/cpa-quiz/`

**Q3：可以隱藏起來不讓別人看到嗎？**
- 免費帳號做不到。GitHub Pages 的 Public repo 任何人有網址都能看
- 想要私密的話：付費 GitHub Pro，或改用其他靜態網站服務（Vercel / Netlify 也免費）

**Q4：可以綁我自己的網域嗎？**
- 可以。在 Settings → Pages 最下方有「Custom domain」欄位
- 需要你已經有一個網域，再去 DNS 設定 CNAME 指向 `你的帳號.github.io`
